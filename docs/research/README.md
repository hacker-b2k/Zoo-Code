# 🔬 Research Log

> Every task's research goes here, one file per topic.
> This index tracks all research conducted in this workspace.

---

## Purpose

Before any design or implementation, research is conducted and documented here.
This builds institutional knowledge and prevents repeating investigations.

## How to Use

1. For a new task, create `docs/research/<topic-slug>.md`
2. Use the template below
3. Add an entry to the index table at the bottom of this file

---

## Research Document Template

```markdown
# Research: <Topic>
Date: YYYY-MM-DD
Status: In Progress | Complete
Related Task: <link to plan>

## Question
What exactly are we trying to learn/solve?

## Current State
What does the code/behavior look like today? (with file references)

## Findings
- Finding 1 (with evidence: file path, line, command output)
- Finding 2
...

## Constraints Discovered
- Constraint 1
- Constraint 2

## Existing Patterns
How does the codebase already solve similar problems?

## Risks & Unknowns
- Risk 1
- Unknown 1 (needs further investigation)

## Conclusion
Summary of what we now know and recommended direction.
```
